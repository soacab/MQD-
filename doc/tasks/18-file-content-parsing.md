# 18 文件内容解析（P2 暂缓）

## 目标
在主流程和文件存在性检查稳定后，支持 Word、PPT、PDF、Excel 等文件内容解析和 file_content 自动检查。

## 参考文档
- `CheckFlow 系统架构设计 v1.md`：FileParser、PromptBuilder、ModelInvoker。
- `CheckFlow 状态机设计 v0 1 38e75a5befd780dd9656d2f830f35f3e.md`：file_content 状态流转。
- `CheckFlow 数据库设计 v1 0.md`：file_parse_jobs。

## 前置依赖
- [ ] 16 文件存在性自动检查完成。

## 最小任务清单
- [ ] T01：启用 `file_parse_jobs` 业务流程；验收：解析任务可创建、排队、成功、失败。
- [ ] T02：实现文件下载与类型识别；验收：不支持类型进入人工处理。
- [ ] T03：接入基础解析器；验收：Word、PPT、PDF、Excel 至少各有一个可运行解析路径。
- [ ] T04：实现解析结果缓存；验收：同文件版本重复检查不重复解析。
- [ ] T05：实现 file_content 判断器；验收：可根据执行规则快照生成初判和依据。
- [ ] T06：处理下载失败、解析失败、前置文件未找到、多候选等分支；验收：状态流转符合状态机。
- [ ] T07：在前端展示解析依据；验收：工程师可查看来源文件和判断片段后确认。

## 验收标准
- [ ] 可对明确目标文件执行内容解析。
- [ ] 解析失败不阻断人工确认。
- [ ] 自动判断只作为初判，不写最终结论。

## 注意事项
本模块标记为 P2 暂缓；P0/P1 未完成前不要投入实现。
