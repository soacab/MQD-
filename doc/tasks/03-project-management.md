# 03 项目管理

## 目标
完成项目基础信息、项目详情、加单、软删除和项目列表能力，为创建点检任务提供项目上下文。

## 参考文档
- `企业办公_点检AI方案设计.md`：4.2、4.3、4.7 项目相关规则。
- `CheckFlow 后端 API 设计 v0 1.md .md`：Project API。
- `CheckFlow 领域模型设计 v1 0.md`：Project Domain。

## 前置依赖
- [ ] 01 数据库迁移与 ORM 完成。
- [ ] 02 用户、认证与权限完成。

## 最小任务清单
- [ ] T01：实现项目列表接口；验收：普通列表默认过滤 `deleted` 项目。
- [ ] T02：实现项目详情接口；验收：返回基础信息、VDrive 信息、加单记录和机型。
- [ ] T03：实现创建项目接口；验收：必填字段缺失时返回业务错误。
- [ ] T04：实现编辑项目基础信息接口；验收：可更新客户、BU、等级、人员、产线等字段。
- [ ] T05：实现项目加单接口；验收：新增接收批次和机型，不覆盖历史记录。
- [ ] T06：实现项目软删除接口；验收：需输入项目名称确认，历史任务和报告保留。
- [ ] T07：实现项目列表、详情、加单、删除确认前端；验收：权限不足时删除入口不可操作。
- [ ] T08：写入项目创建、加单、删除审计日志；验收：关键动作可追溯。

## 验收标准
- [ ] 可以创建、查询、编辑项目。
- [ ] 可以保存多个项目接收批次和机型。
- [ ] 删除项目后普通列表不展示。
- [ ] `deleted` 项目不能继续创建点检任务。

## 注意事项
项目删除为软删除，不物理删除项目、任务、报告、审计数据。
