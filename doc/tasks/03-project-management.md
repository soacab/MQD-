# 03 项目管理

## 目标
完成内部 Project 聚合对象、项目档案详情、加单、软删除和项目列表能力，为新建点检任务提供历史上下文。用户侧不再存在独立“创建项目”入口。

## 当前状态
🟡 P0 项目档案维护能力已补强。Project 保留为后端内部聚合和档案对象，用于承载项目基础信息、VDrive 标识、历史任务、报告和加单记录；普通用户不再单独创建项目。后端仍保留项目增删改查、加单、VDrive 标识保存、软删除阻断、列表筛选、`deleted` 管理查询和加单前置状态校验，`POST /api/v1/projects` 仅作为兼容/后台数据维护能力。前端 `/projects` 已收敛为项目档案维护页，只用于查询、编辑历史基础信息、更新 VDrive、加单和作废。新建点检任务入口已支持 VDrive 校验后回填历史项目信息、无历史时使用 VDrive 返回文件夹名作为项目名称候选，并推荐 QG 节点。

## 验收标准
- [x] 普通前端不暴露独立创建项目入口；项目档案由新建点检任务时自动创建或复用。
- [x] 可以在前端查询、编辑项目档案，并覆盖方案要求的项目基础字段。
- [x] 可以保存多个项目接收批次和机型。
- [x] 删除项目后普通列表不展示。
- [x] `deleted` 项目不能继续创建点检任务。
- [x] 删除项目前端存在真实二次确认，且必须手动输入项目名称。
- [x] VDrive 校验后支持历史信息回填和 QG 节点推荐。
- [x] 加单遵守无进行中任务、无复查中任务的业务前置条件。

## 注意事项
项目删除为软删除，不物理删除项目、任务、报告、审计数据。
