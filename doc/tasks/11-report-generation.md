# 11 报告生成

## 目标
归档时创建或更新节点报告，并保存检查项过程记录，用于档案查询和后续更正。

## 参考文档
- `企业办公_点检AI方案设计.md`：4.5、4.7 报告相关规则。
- `CheckFlow 领域模型设计 v1 0.md`：Report Domain。
- `CheckFlow 数据库设计 v1 0.md`：报告域表。

## 前置依赖
- [x] 10 归档当前轮完成。

## 最小任务清单
- [x] T01：在创建任务时创建 `inspection_reports` 初始记录；验收：一个任务只有一份报告。
- [x] T02：归档时更新报告综合结论；验收：报告结论与归档节点结论一致。
- [x] T03：创建或更新 `report_items`；验收：每个检查项按 `source_rule_code` 汇总一条报告明细。
- [x] T04：写入 `process_records_json`；验收：记录轮次、时间、结论、判定人和说明。
- [x] T05：实现报告列表接口；验收：支持项目、工程师、QG 节点、结论、时间筛选。
- [x] T06：实现报告详情接口；验收：返回项目明细、规则版本、综合结论和检查项明细。
- [x] T07：实现报告详情页；验收：可按 QG 节点查看报告和过程记录。
- [x] T08：实现待跟进落实时间在报告中展示；验收：关闭待跟进项后报告可见实际完成时间。

## 验收标准
- [x] 归档后可查看报告。
- [x] 同一检查项可展示多轮过程记录。
- [x] 报告展示任务创建时的规则快照。
- [x] 报告是否更正由更正记录推导，不单独存状态。

## 注意事项
InspectionReport 属于 InspectionTask，不属于某一轮 InspectionRound。
