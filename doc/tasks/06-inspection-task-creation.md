# 06 创建点检任务

## 目标
实现从 VDrive 链接、关联项目基础信息和已发布规则新建点检任务的唯一用户入口，并完成创建前业务校验。

## 当前状态
✅ 新建点检任务已按业务方案对齐为 VDrive-first 两步向导。用户侧没有独立“创建项目”动作；Project 仅作为后端内部聚合和档案对象，由新建点检任务时自动复用或创建。`POST /api/v1/inspection-tasks/prepare` 用于校验 VDrive 链接、回填历史项目信息并推荐 QG 节点；无历史记录时使用 VDrive 返回的文件夹名作为项目名称候选。`POST /api/v1/inspection-tasks` 同时兼容旧的 `project_id + qg_node_id` 创建和新的向导式创建。

## 最小任务清单
- [x] T01：实现 `POST /api/v1/inspection-tasks` 请求模型；验收：包含关联项目基础信息、QG 节点、创建人上下文。
- [x] T02：校验项目 `status = normal`；验收：作废项目创建任务失败。
- [x] T03：校验项目已保存 VDrive 文件夹标识；验收：缺少 VDrive 信息时返回业务错误。
- [x] T04：查询当前 QG 节点 published 规则版本；验收：无已发布版本时不能创建任务。
- [x] T05：校验同项目同节点无 running / rectifying 任务；验收：重复活跃任务被阻断。
- [x] T06：创建 `inspection_tasks` 主记录；验收：初始状态为 `running`。
- [x] T07：实现新建任务前端两步表单；验收：必填信息完整后才能确认创建。
- [x] T08：创建任务后跳转点检执行页；验收：前端可进入当前任务详情。
- [x] T09：实现任务作废接口；验收：`running` 或 `rectifying` 任务可填写原因后变为 `voided`，后续确认、归档、整改、复查入口全部禁用。
- [x] T10：串联创建任务完整业务事件；验收：`POST /api/v1/inspection-tasks` 最终在同一事务内完成任务主记录、规则快照、首轮、检查项和报告初始化，任一步失败不得残留半成品。
- [x] T11：实现任务创建预检接口；验收：VDrive 校验通过后返回 VDrive 识别结果、历史项目信息和推荐 QG 节点。
- [x] T12：实现向导式任务创建；验收：可从 VDrive 链接、关联项目基础信息、接收时间、机型和 QG 节点一次性创建或复用内部 Project，并创建 InspectionTask。

## 验收标准
- [x] 可以基于 normal 项目和 published 规则创建点检任务。
- [x] 可以从 VDrive 链接发起新建点检任务，无需用户手动输入项目 ID。
- [x] 用户侧只存在新建点检任务入口，不存在独立创建项目入口。
- [x] 无历史记录时使用 VDrive 返回文件夹名作为项目名称候选；有历史记录时回填项目基础信息和机型，并推荐下一 QG 节点。
- [x] 关键前置条件失败时有明确错误。
- [x] 创建动作写入审计日志。
- [x] 作废任务必须保留历史数据，写入作废原因和审计日志。
- [x] 已作废任务不能再执行确认、归档、整改、复查或报告更正。

## 注意事项
本模块负责唯一创建入口和校验；规则快照、轮次、检查项和报告初始化由后续模块实现，但最终必须被创建任务业务事件原子串联。当前点检页按“流程契约先行”验证本系统任务、检查项确认、归档、报告、整改复查闭环；VDrive 扫描、文件内容解析、QMS/UCM 直连仍为 mock 或未接入真实接口，不作为真实点检有效性结论。
