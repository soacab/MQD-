# 12 整改项与待跟进项

## 目标
管理 NO_GO 产生的整改项和 C_GO 产生的待跟进项，支持标记完成、撤销和落实。

## 参考文档
- `企业办公_点检AI方案设计.md`：4.2、4.6 整改追踪与待跟进。
- `CheckFlow 状态机设计 v0 1 38e75a5befd780dd9656d2f830f35f3e.md`：RectificationItem、FollowUpItem。
- `CheckFlow 后端 API 设计 v0 1.md .md`：Rectification API、Follow-up API。

## 前置依赖
- [x] 10 归档当前轮完成。
- [x] 11 报告生成完成。

## 最小任务清单
- [x] T01：归档 NO_GO 时生成整改项；验收：每个 fail 检查项生成一条整改项。
- [x] T02：归档 C_GO 时生成待跟进项；验收：每个 conditional 检查项生成一条待跟进项。
- [x] T03：实现整改项列表接口；验收：可按任务、项目、责任人、超期状态查询。
- [x] T04：实现标记整改完成接口；验收：写入标记人和标记时间。
- [x] T05：实现撤销整改完成接口；验收：复查触发前可清空标记信息。
- [x] T06：实现待跟进项列表和关闭接口；验收：关闭后记录实际完成时间并移出待办。
- [x] T07：实现整改追踪页；验收：可批量勾选、二次确认、查看整改通知。
- [x] T08：实现待跟进入口；验收：工作台可展示超期待跟进并进入落实弹窗。

## 验收标准
- [x] NO_GO 后能看到整改项。
- [x] C_GO 后能看到待跟进项。
- [x] 整改完成可撤销，复查后不可撤销。
- [x] 待跟进关闭不改变任务 completed 状态。

## 注意事项
整改执行发生在系统外；系统只记录工程师标记和复查触发条件。
