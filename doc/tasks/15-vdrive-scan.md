# 15 VDrive 候选文件识别

## 目标
接入 VDrive 文件元数据读取能力，为每个检查项识别候选文件，并保存候选文件记录。

- `CheckFlow 系统架构设计 v1.md`：VDrive Adapter。

## 当前状态

🟡 后端 Adapter/API 基础已具备。VDrive Adapter 已按真实接口文档形成 `folderGuid -> folderId` 和文件/文件夹元数据读取边界；无真实 VDrive 配置时继续使用 Mock Adapter。检查项级候选文件扫描、列表、选择和重试 API 已接入，候选文件会写入 `auto_check_candidate_files`。

file_existence 自动初判缺口合并到本模块继续跟进：真实扫描仍需手动调用 scan/retry API，任务创建和复查轮生成后尚未自动切到真实扫描；前端候选文件确认、自动检查依据展示、自动检查关闭降级和工程师最终确认闭环尚未完成。

## 最小任务清单
- [x] T01：实现检查项候选文件识别接口；验收：可按检查项读取 VDrive 文件元数据并返回候选文件。
- [x] T02：实现 VDrive 文件元数据 Adapter；验收：可读取项目根目录下相关文件信息。
- [x] T03：保存 `auto_check_candidate_files`；验收：文件名称、版本、创建时间、大小、路径和推荐原因可查询。
- [x] T04：处理候选文件为空、多候选和 VDrive 异常；验收：检查项可进入待选文件或人工判断状态。
- [ ] T05：实现前端候选文件确认；验收：用户可选择候选文件或改为人工判断。
- [ ] T06：将任务创建和复查轮生成后的 file_existence 检查从 mock / 手动扫描切到真实扫描入口；验收：符合条件的自动检查项不需要人工点击即可进入检查链路。
- [ ] T07：实现自动检查依据前端展示和工程师最终确认闭环；验收：自动检查只提供初判，最终结论仍由工程师提交。
- [ ] T08：实现自动检查关闭降级；验收：系统设置关闭自动检查时，pending 自动项可明确进入人工处理路径。

## 验收标准
- [x] 可以按检查项识别 VDrive 候选文件。
- [x] 可以保存与检查项判断相关的候选文件。
- [x] VDrive 异常不影响项目历史数据。
- [x] 候选文件可被自动检查结果和工程师确认引用。
- [ ] 创建任务和触发复查后，符合条件的 file_existence 自动检查项可自动进入真实扫描链路。
- [ ] 自动检查不直接产生最终结论，工程师仍需最终确认。

## 注意事项
一期不保存完整 VDrive 目录快照；若未来需要追溯整个目录现场，再单独设计目录快照能力。
