# 15 VDrive 候选文件识别

## 目标
接入 VDrive 文件元数据读取能力，为每个检查项识别候选文件，并保存候选文件记录。

## 参考文档
- `CheckFlow MVP 开发任务拆分 v1 0 38e75a5befd780778d1ef8bdd4cd1e65.md`：Phase 6。
- `CheckFlow 系统架构设计 v1.md`：VDrive Adapter。
- `CheckFlow 数据库设计 v1 0.md`：自动检查候选文件表。

## 前置依赖
- [ ] 04 VDrive 链接校验完成。
- [ ] 08 生成轮次和检查项完成。

## 最小任务清单
- [ ] T01：实现检查项候选文件识别接口；验收：可按检查项读取 VDrive 文件元数据并返回候选文件。
- [ ] T02：实现 VDrive 文件元数据 Adapter；验收：可读取项目根目录下相关文件信息。
- [ ] T03：保存 `auto_check_candidate_files`；验收：文件名称、版本、创建时间、修改时间、大小、路径和推荐原因可查询。
- [ ] T04：处理候选文件为空、多候选和 VDrive 异常；验收：检查项可进入待选文件或人工判断状态。
- [ ] T05：实现前端候选文件确认；验收：用户可选择候选文件或改为人工判断。

## 验收标准
- [ ] 可以按检查项识别 VDrive 候选文件。
- [ ] 可以保存与检查项判断相关的候选文件。
- [ ] VDrive 异常不影响项目历史数据。
- [ ] 候选文件可被自动检查结果和工程师确认引用。

## 注意事项
一期不保存完整 VDrive 目录快照；若未来需要追溯整个目录现场，再单独设计目录快照能力。
