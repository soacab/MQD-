# 07 生成规则快照

## 目标
在点检任务创建时冻结业务规则和自动检查执行规则，确保进行中任务不受后续规则发布影响。

## 参考文档
- `CheckFlow 领域模型设计 v1 0.md`：快照优先原则。
- `CheckFlow 数据库设计 v1 0.md`：`rule_snapshots` 表。
- `CheckFlow 系统架构设计 v1.md`：创建点检任务时序。

## 前置依赖
- [ ] 06 创建点检任务完成。

## 最小任务清单
- [ ] T01：设计业务规则快照 JSON 结构；验收：包含任务展示和报告引用所需字段。
- [ ] T02：设计自动检查执行规则快照 JSON 结构；验收：包含执行模式、Adapter、配置版本和 `config_json`。
- [ ] T03：实现 RuleService 生成快照方法；验收：输入 published 规则版本可返回两类快照。
- [ ] T04：创建 `rule_snapshots` 记录并绑定任务；验收：`inspection_task_id` 唯一。
- [ ] T05：在创建任务事务中写入快照；验收：任务创建失败时快照不残留。
- [ ] T06：添加快照读取接口或服务方法；验收：后续生成检查项可只依赖快照。

## 验收标准
- [ ] 任务创建时生成规则快照。
- [ ] 后续修改规则不影响已有任务快照。
- [ ] 快照中业务规则和自动执行规则分层保存。

## 注意事项
快照是任务级事实，不能在点检过程中被规则配置页面回写修改。
