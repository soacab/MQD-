# 10 归档当前轮

## 目标
在所有检查项完成后归档当前轮，计算节点结论并推进任务状态。

## 前置依赖
- [ ] 09 工程师确认检查项完成。

## 最小任务清单
- [ ] T01：实现归档前置校验；验收：任务和当前轮必须为 `running`。
- [ ] T02：校验所有检查项已 `confirmed` 或 `inherited`；验收：存在待处理项时归档失败。
- [ ] T03：实现节点结论计算；验收：fail 优先为 NO_GO，其次 conditional 为 C_GO，否则 FULL_GO。
- [ ] T04：更新当前轮状态和归档时间；验收：`inspection_rounds.status = archived`。
- [ ] T05：根据结论更新任务状态；验收：NO_GO 进入 `rectifying`，C_GO/FULL_GO 进入 `completed`。
- [ ] T06：生成整改项和待跟进项触发点；验收：fail 生成整改项，conditional 生成待跟进项。
- [ ] T07：实现归档弹窗；验收：展示本轮摘要、不满足项、带条件满足项和确认按钮。
- [ ] T08：写入归档审计日志；验收：归档人、时间、结论可追溯。

## 验收标准
- [ ] 所有检查项确认后可归档。
- [ ] FULL_GO、C_GO、NO_GO 三种结果均可跑通。
- [ ] 归档是状态推进唯一入口。

## 注意事项
归档是关键业务边界；不要提供通用状态更新接口绕过归档动作。
