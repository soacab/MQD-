对抗审查确认的待修复问题：

- Critical：业务数据读取和写入接口缺少范围校验。需要按方案 4.1 实现 `inspection_engineer` 本人范围、`project_admin` 全量范围、权限取并集、权限管理无业务范围。
- Important：用户列表必须始终排除 `deleted` 账号；当前仍需补显式 `status=deleted` 和权限筛选组合下的删除账号不可见验证。
- Important：后台页还未完全对齐原型的后台管理信息架构；需要补左侧“用户与权限 / 系统设置”导航、中间账号列表和右侧权限说明布局。
- Important：前端需本地拦截当前登录用户停用、删除、取消个人权限管理权限，并在编辑当前用户后同步本地会话快照。
- Important：测试覆盖不足。需补普通账号启停用登录效果、已有 token 被停用后拒绝、审计日志断言、前端行为测试和业务越权测试。
- Important：README 默认后端测试命令尚未覆盖全部 `backend/tests`；`uv run pytest` 当前不是可靠默认入口。

## 验收标准
- [x] UID 登录流程可跑通。
- [x] 当前用户接口可返回权限集合。
- [x] 后端权限支持叠加，不互相收窄。
- [x] 后端至少保留一个启用的权限管理员。
- [x] 历史数据不因账号停用或删除而丢失。
- [ ] 前端可维护同一用户的多权限，并通过行为测试验证。
- [ ] 前端可启用、停用账号，并展示明确状态变化；当前用户不可停用。
- [ ] 前端可删除访问配置，且不能删除当前用户或最后一个权限管理员。
- [ ] 无权限用户进入后台时只读或不可操作，并有明确提示；需行为测试覆盖。
- [ ] 权限维护、账号启停用、删除访问配置和审计日志有专项测试覆盖。
- [ ] 业务数据访问范围符合方案 4.1，不因“已登录”或“权限管理”而获得业务数据范围。
