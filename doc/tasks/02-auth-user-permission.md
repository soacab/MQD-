# 02 用户、认证与权限

## 目标
完成 UID 登录、用户权限维护、会话用户查询和系统设置基础能力。

## 前置依赖
- [x] 00 项目初始化完成。
- [x] 01 数据库迁移与 ORM 完成。

## 最小任务清单
- [x] T01：实现 `POST /api/v1/auth/login`；验收：有效 UID 可登录，停用 UID 被阻断。
- [x] T02：实现 `GET /api/v1/auth/me`；验收：返回当前用户与权限集合。
- [x] T03：实现用户列表、创建用户接口；验收：可按 UID、姓名、邮箱、权限、状态查询。
- [x] T04：实现用户权限更新接口；验收：同一 UID 可配置多个权限。
- [x] T05：实现启用、停用、删除访问配置；验收：不能停用或删除最后一个权限管理员。
- [x] T06：实现系统设置读取与保存；验收：自动检查开关可持久化并记录保存人。
- [x] T07：实现前端登录页和后台账号管理页；验收：有权限用户可管理账号，无权限用户只读或不可操作。
- [x] T08：写入权限变更审计日志；验收：新增、停用、权限变更均可追溯。

## 验收标准
- [x] UID 登录流程可跑通。
- [x] 权限支持叠加，不互相收窄。
- [x] 至少保留一个启用的权限管理员。
- [x] 历史数据不因账号停用或删除而丢失。

## 注意事项
CheckFlow 不维护公司账号密码；MVP 可使用本地密码校验或模拟认证，但接口边界需保留公司账号体系接入点。
