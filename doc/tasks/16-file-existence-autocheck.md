# 16 文件存在性自动检查

## 目标
基于 VDrive 扫描快照和自动检查执行规则，完成 file_existence 类型检查项的自动初判。

## 最小任务清单
- [ ] T01：实现自动检查启动接口和调度入口；验收：file_existence 检查项可由任务创建、复查轮生成或手动重试触发自动检查。
- [ ] T02：读取检查项、执行规则快照和最新扫描批次；验收：缺少任一输入时进入人工处理。
- [ ] T03：实现候选文件匹配逻辑；验收：按文件名、路径、版本、时间等配置筛选候选。
- [ ] T04：保存 `auto_check_results`；验收：记录执行状态、初判结果、依据和错误信息。
- [ ] T05：保存 `auto_check_candidate_files`；验收：多候选时可展示候选列表。
- [ ] T06：处理明确找到文件；验收：检查项进入 `auto_completed`，等待工程师确认。
- [ ] T07：处理未找到、多候选、VDrive 异常；验收：分别进入 `manual_required` 或 `candidate_waiting`。
- [ ] T08：实现候选文件选择和重试接口；验收：选择候选后可重新自动检查。
- [ ] T09：实现自动检查结果前端展示；验收：工程师能查看依据并提交最终确认。
- [ ] T10：实现自动检查关闭降级；验收：系统设置关闭自动检查时，pending 自动项可明确进入人工处理路径。

## 验收标准
- [ ] file_existence 检查项可自动检查。
- [ ] 未找到、多候选、明确找到、扫描失败四种结果可区分。
- [ ] 自动检查不直接产生最终结论。
- [ ] 工程师仍需最终确认。
- [ ] 创建任务和触发复查后，符合条件的自动检查项不需要人工点击即可进入检查链路。

## 注意事项
单项自动检查异常不阻断其他检查项执行。
