# 05 规则配置与版本管理

## 目标
实现 QG 节点、业务检查项、自动检查执行规则和规则版本发布能力。

## 参考文档
- `企业办公_点检AI方案设计.md`：4.8 规则配置与版本管理。
- `CheckFlow 后端 API 设计 v0 1.md .md`：Rule API、Auto Check Execution Rule API。
- `CheckFlow 领域模型设计 v1 0.md`：Rule Domain。

## 前置依赖
- [ ] 01 数据库迁移与 ORM 完成。
- [ ] 02 用户、认证与权限完成。

## 最小任务清单
- [ ] T01：实现 QG 节点列表接口；验收：前端可展示 QG2 至 QG4 节点。
- [ ] T02：实现规则版本创建、列表、详情接口；验收：可维护 draft、published、deprecated 状态。
- [ ] T03：实现业务检查项新增与编辑接口；验收：字段覆盖名称、类型、Checklist、责任方、APQP、排序、启用状态。
- [ ] T04：实现自动检查执行规则配置接口；验收：自动项和系统直连项可绑定执行规则。
- [ ] T05：实现规则发布校验；验收：自动项缺少启用执行规则时发布失败。
- [ ] T06：实现发布规则版本动作；验收：新版本 published 后旧版本自动 deprecated。
- [ ] T07：实现规则配置前端页面；验收：规则管理员可编辑发布，其他权限只读查看。
- [ ] T08：实现版本历史与变更说明展示；验收：可查看版本号、发布人、发布时间、变更内容。

## 验收标准
- [ ] 可以创建规则版本。
- [ ] 可以新增和编辑检查项。
- [ ] 可以配置自动检查执行规则。
- [ ] 已发布版本可用于创建点检任务。

## 注意事项
业务规则用于展示和生成检查项；自动执行规则用于后端真实执行，两者不能混为一张表。
