# 14 报告更正

## 目标
支持授权用户对报告单项结论追加更正记录，并重新计算展示结论。

## 参考文档
- `企业办公_点检AI方案设计.md`：4.7 检查档案与历史报告。
- `CheckFlow 状态机设计 v0 1 38e75a5befd780dd9656d2f830f35f3e.md`：CorrectReportItem。
- `CheckFlow 后端 API 设计 v0 1.md .md`：Report API。

## 前置依赖
- [ ] 11 报告生成完成。

## 最小任务清单
- [ ] T01：实现报告更正权限校验；验收：仅本任务点检执行工程师本人、`project_admin` 或 `super_admin` 可更正。
- [ ] T02：实现 `POST /api/v1/reports/{report_id}/items/{report_item_id}/correct`；验收：必须填写更正说明。
- [ ] T03：写入 `report_corrections`；验收：保存修改前结论、修改后结论、修改人、修改时间和说明。
- [ ] T04：更新 `report_items` 当前展示结论；验收：报告详情显示人工修正标签。
- [ ] T05：重新计算报告综合结论；验收：单项更正影响 FULL_GO / C_GO / NO_GO 时报告展示同步变化。
- [ ] T06：实现更正弹窗；验收：展示修正前状态、说明输入框和最终结论选择。
- [ ] T07：写入报告更正审计日志；验收：更正动作可追溯。

## 验收标准
- [ ] 授权用户可更正报告单项结论。
- [ ] 更正必须追加记录，不覆盖原始点检过程。
- [ ] 更正后报告综合结论可重新计算。
- [ ] 更正不修改 `InspectionItem`、不重新生成整改项或待跟进项、不改变 `InspectionTask.status`。

## 注意事项
报告更正不修改 AI 原始结论、工程师原始确认记录、检查项实例和归档时过程记录，只影响报告当前展示结论。
