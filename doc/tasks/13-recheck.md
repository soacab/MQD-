# 13 触发复查

## 目标
在整改项全部完成后创建下一轮点检，只复查上一轮不满足项，并保留多轮报告记录。

## 参考文档

- `CheckFlow 状态机设计 v0 1 38e75a5befd780dd9656d2f830f35f3e.md`：TriggerRecheck。
- `CheckFlow 系统架构设计 v1.md`：整改复查时序。

## 前置依赖
- [ ] 12 整改项与待跟进项完成。

## 最小任务清单
- [ ] T01：实现复查前置校验；验收：任务必须为 `rectifying`，最新轮必须已归档。
- [ ] T02：校验所有整改项已完成；验收：存在未完成整改项时不能触发复查。
- [ ] T03：创建下一轮 `inspection_rounds`；验收：`round_no = current_round_no + 1`。
- [ ] T04：更新任务状态和当前轮号；验收：任务回到 `running`。
- [ ] T05：根据上一轮 fail 项生成复查检查项；验收：历史已通过项不重复生成。
- [ ] T06：保留历史已通过项用于报告展示；验收：报告可展示历史带入信息。
- [ ] T07：实现触发复查确认弹窗；验收：展示本轮复查项数量和历史带入项数量。
- [ ] T08：写入触发复查审计日志；验收：复查轮创建动作可追溯。
- [ ] T09：触发复查后投递自动检查；验收：复查轮中符合 `file_existence` 且有启用执行规则快照的 `pending` 项进入自动检查链路。

## 验收标准
- [ ] 所有整改项完成后可触发复查。
- [ ] 复查轮只生成上一轮 fail 项。
- [ ] 复查轮可继续确认、归档。
- [ ] 报告能看到多轮过程记录。
- [ ] 自动检查关闭或未启用时，复查检查项可明确转人工处理。

## 注意事项
复查复用点检执行模块，不另建一套确认逻辑。
