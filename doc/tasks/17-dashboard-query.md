# 17 工作台与查询

## 目标
提供用户登录后的工作台概览和待办入口，让工程师能快速进入任务、整改、跟进和报告。

## 参考文档
- `企业办公_点检AI方案设计.md`：4.2 任务状态管理。
- `CheckFlow 数据库设计 v1 0.md`：Dashboard API。
- `CheckFlow 后端 API 设计 v0 1.md .md`：Dashboard API。

## 前置依赖
- [ ] 10 归档当前轮完成。
- [ ] 12 整改项与待跟进项完成。
- [ ] 16 文件存在性自动检查完成。

## 最小任务清单
- [ ] T01：实现 `GET /api/v1/dashboard/overview`；验收：返回进行中、整改中、待跟进、待归档数量。
- [ ] T02：实现 `GET /api/v1/dashboard/my-todos`；验收：返回当前用户可处理的待办。
- [ ] T03：实现进行中任务查询；验收：展示自动检查状态和检查项进度。
- [ ] T04：实现复查中任务查询；验收：展示整改进度和触发复查入口。
- [ ] T05：实现待跟进查询；验收：超期项标记为超期并显示计划完成时间。
- [ ] T06：实现候选文件待选择和待归档查询；验收：可从工作台跳转到对应检查项或任务。
- [ ] T07：实现工作台前端三分区布局；验收：进行中、复查中、待跟进互不混排。

## 验收标准
- [ ] 用户进入系统后能看到当前待办。
- [ ] 待办可跳转到任务、检查项、整改项或报告。
- [ ] 超期整改和超期待跟进能正确推导。

## 注意事项
可推导状态不入库；工作台展示状态应从事实字段实时计算。
